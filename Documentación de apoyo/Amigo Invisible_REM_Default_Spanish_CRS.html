1 Introducción

Describiremos el flujo de actividades llevadas a cabo que nuestra aplicación deberá satisfacer, con lo que ello nos llevará a la identificación de los requisitos de la aplicación y todos los elementos del documento de requisitos del sistema.
La actividad llevada a cabo será una aplicación para realizar las funciones de preparación del amigo invisible. En nuestro caso dicha actividad seguirá los siguientes pasos:
Introduciremos el evento(nombre, dia, hora, lugar de celebración,...), el cual deberá tener al menos unos valores introducidos para nombre, lugar, hora y fecha. Tendremos la opción al entrar en el nuevo evento a que carguemos los datos de un evento anterior que no ha sido realizado al completo por comodidad. Seleccionaremos un grupo de participantes introducidos previamente, dicho grupo podrá ser modificado y eliminado. En cada grupo habrá un número determinado de participantes que podrán ser añadidos manualmente o mediante la agenda del teléfono, eliminados y modificados. Tras seleccionar el grupo de personas incluidas en el juego, avanzaremos a una pantalla en la cual marcaremos si hay o no exclusiones entre los participantes. Si las hubiese, marcariamos la casilla correspondiente y las incluiriamos, pudiendo eliminar exclusiones hechas con anterioridad o pudiendo marcar la casilla doble sentido en la cual realizaremos una exclusión hacia los dos participantes elegidos. Con todo esto realizado mostrará por pantalla todos los datos del evento y los participantes elegidos para comprobar los datos, aquí podremos seleccionar la realización del sorteo y si se querrá que los participantes reciban SMS individualmente a parte del correo electrónico por defecto(el correo electrónico es obligatorio introducirlo para todos los participantes). En este punto los participantes recibirán las notificaciones oportunas sobre los datos del evento y a quién tendrán que ir destinados sus regalos.

    UC-0006 Añadir Grupo
    Versión 1.0 ( 30/12/2013 )
    Autores
    • Quintín Sanabria Sánchez
    Fuentes ?
    Dependencias Ninguno
    Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el usuario pulse sobre "Crear Grupo" y desee añadir un grupo de participantes para el evento.
    Precondición Haber introducido un evento valido en "Introducir Evento"
    Secuencia normal Paso Acción
    1 El actor Usuario (ACT-0001) seleccionará el botón "Crear Grupo" e introducirá el nombre del grupo de participantes a crear.
    2 Si el usuario pulsa "Continuar", se realiza el caso de uso Comprobar Nombre Grupo (UC-0020)
    3 Si la verificación del nombre es correcta, el sistema continua la ejecución
    4 Si el usuario desea añadir participantes al grupo, se realiza el caso de uso Añadir Participante (UC-0008)
    5 Si el usuario desea borrar todos los participantes añadidos pulsa "Limpiar Lista", se realiza el caso de uso Limpiar Lista (UC-0030)
    6 Si el usuario pulsa el botón "Agregar", se realiza el caso de uso Guardar Grupo (UC-0014)
    7 Si el guardado del grupo es correcto, el sistema finaliza el caso de uso correctamente.
    Postcondición PD
    Excepciones Paso Acción
    2 Si el usuario pulsa "Cancelar", el sistema en este caso de uso no revertirá ningún efecto, a continuación este caso de uso queda sin efecto
